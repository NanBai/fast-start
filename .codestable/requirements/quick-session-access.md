# 快速找到并启动高频工作现场

## 用户故事

- 作为每天在多个项目之间切换的人，我希望输入几个关键词就找到目标 session，而不是展开 agent 和项目一层层翻。
- 作为长期反复回到同一批项目的人，我希望把这些项目固定在前面，而不是每次都被最近活跃时间重新挤下去。
- 作为正在恢复上下文的人，我希望用键盘就能定位并启动目标 session，而不是在列表里来回移动鼠标。

## 为什么需要

Session Launcher 的价值是快速恢复工作现场。当前列表已经能按 agent、项目和最近天数整理，但 session 一多，用户还是要靠展开、扫标题、找路径来定位。这个成本会直接抵消“一键恢复”的速度优势。

## 怎么解决

给列表加一个快速定位入口：用户输入项目名、路径、简介或 agent 名就能缩小结果；常用项目可以收藏并排在更靠前的位置；焦点在搜索入口时可以用键盘选择并启动匹配项。

## 边界

- 不负责改进 CLI 的扫描来源，也不改变 session 恢复命令。
- 不做跨设备同步，收藏只保存在本机。
- 不做单条 session 收藏；本轮先以项目目录作为稳定收藏对象。
- 不做全局系统快捷键；快捷键只在 app 窗口获得焦点时生效。
- 不替代最近天数筛选，搜索只在当前时间范围内继续收窄结果。
